## Skill trigger: `postgresql-performance-check`

Invoke the `postgresql-performance-check` skill when the user reports or asks about:
- Slow queries or high query latency
- Database load or high CPU from PostgreSQL
- Lock contention, deadlocks, or blocked queries
- Cache hit ratio or shared_buffers efficiency
- Vacuum issues (autovacuum not running, table bloat from missed vacuums)
- Connection saturation or "too many connections" from application load
- pg_stat_statements analysis or top queries by time/calls
- Long-running transactions or idle-in-transaction sessions causing issues
