## Skill trigger: `postgresql-connection-check`

Invoke the `postgresql-connection-check` skill when the user reports or asks about:
- PgBouncer or connection-pooler issues
- "too many connections" or max_connections exhaustion
- Connection leaks or idle-in-transaction sessions
- Application unable to connect to the database
- Service endpoint resolution issues (DNS, Service, Endpoints)
- Pool utilization or PgBouncer stats
- Connection refused errors from PostgreSQL
- Connection timeout from application to database
